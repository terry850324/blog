Notepad++ 編譯 VB.NET 環境設定

發表於 2020-12-09 更新於 2020-12-10 分類於 吃飽太閒 閱讀次數： Disqus：

非工作用的電腦重灌以後，不想再裝Visual Studio占用電腦空間，想要改用Notepad++編譯VB.NET。設定編譯環境之前，電腦非常乾淨，需要先從安裝VB.NET編譯器開始。

編譯器

下載與安裝

如果沒有安裝Visual Studio，要先到.NET官網下載並安裝.NET SDK，安裝完成後才有VB.NET編譯器，編譯器位置在：

C:\Windows\Microsoft.NET\Framework\v4.0.30319

編譯器執行檔為：vbc.exe（C#編譯器為csc.exe）。

設定環境變數

想辦法找到設定環境變數的地方，通常從桌面找到電腦，點擊右鍵>「內容」>「進階系統設定」>「環境變數」。然後在「Path」中加入編譯器所在目錄。

註：環境變數設定非必要，但是能方便日後下指令。

Notepad++設定

開啟Notepad++，選擇「執行」>「執行」（圖1），在跳出視窗的視窗中輸入指令（圖2），然後儲存方便日後使用。

圖1

圖2

編譯

有設定環境變數
cmd /k vbc.exe/out:"$(CURRENT_DIRECTORY)\$(NAME_PART).exe" "$(FULL_CURRENT_PATH)" & PAUSE & EXIT
無設定環境變數
cmd /k C:\Windows\Microsoft.NET\Framework\v4.0.30319\vbc.exe/out:"$(CURRENT_DIRECTORY)\$(NAME_PART).exe" "$(FULL_CURRENT_PATH)" & PAUSE & EXIT

編譯完執行

有設定環境變數
cmd /k vbc.exe/out:"$(CURRENT_DIRECTORY)\$(NAME_PART).exe" "$(FULL_CURRENT_PATH)" & echo Running: & "$(CURRENT_DIRECTORY)\$(NAME_PART).exe" & PAUSE & EXIT
無設定環境變數
cmd /k C:\Windows\Microsoft.NET\Framework\v4.0.30319\vbc.exe/out:"$(CURRENT_DIRECTORY)\$(NAME_PART).exe" "$(FULL_CURRENT_PATH)" & echo Running: & "$(CURRENT_DIRECTORY)\$(NAME_PART).exe" & PAUSE & EXIT

執行

cmd /k echo Running: & "$(CURRENT_DIRECTORY)\$(NAME_PART).exe" & PAUSE & EXIT

註：有沒有設定環境變數就差在指令長度。如果要編譯C#，把vbc.exe改為csc.exe。

撰寫程式

寫一隻簡單的Hello World程式，然後儲存。

Module Module1
    Sub Main()
        Console.WriteLine("Hello World")
        Console.WriteLine("This is the first VB.Net Application.")
    End Sub
End Module

執行編譯+執行。
